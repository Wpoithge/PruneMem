# Contributing

Thanks for your interest in PruneMem.

## Ground rules

- Do not commit private memory data or real user chat logs.
- Keep provider credentials out of the repository.
- Prefer portable paths and config-driven behavior.
- Add or update tests for lifecycle/governance behavior changes.

## Development flow

1. Fork or branch from `main`
2. Make focused changes
3. Add/update tests and docs
4. Run local checks
5. Open a PR with scope and reasoning

## Pull request expectations

Please explain:
- what changed
- why it changed
- whether it affects schema, adapters, or runtime behavior
- how it was tested
